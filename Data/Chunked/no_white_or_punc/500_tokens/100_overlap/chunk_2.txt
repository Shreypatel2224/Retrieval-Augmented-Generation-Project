most modern file systems Btrees address effectively all of the major problems encountered when implementing diskbased search trees 1 The Btree is shallow in part because the tree is always height balanced all leaf nodes are at the same level and in part because the branching factor is quite high So only a small number of disk blocks are accessed to reach a given record 2 Update and search operations affect only those disk blocks on the path from the root to the leaf node containing the query
